раздел 1

Основная структура HTML-страницы

Любая HTML-страница начинается с объявления типа документа <!doctype html> и корневого элемента <html>. Можно представить, что это «игровой движок», внутри которого есть настройки (<head>) и сам уровень (<body>), где уже «живут» блоки наподобие интерфейса из Minecraft или инвентаря из Stardew Valley. 

<!doctype html>
<html lang="ru">
  <head>
    <meta charset="utf-8">
    <title>Экран выбора персонажа</title>
  </head>

  <body>
    <h1>Выберите героя для приключения</h1>
    <p>Например, маг как в Baldur's Gate или отважный шахтёр как в Minecraft.</p>
  </body>
</html>
РЕЗУЛЬТАТ:

Выберите героя для приключения

Например, маг как в Baldur's Gate или отважный шахтёр как в Minecraft.

Важно: <meta charset="utf-8"> нужен, чтобы русские буквы корректно отображались и имена вроде «Айзек» или «Деспот» не разваливались в символы.
раздел 2

Заголовки и текст

Заголовки H1–H6

Заголовки можно воспринимать как уровни меню в игре: главный логотип, названия разделов, подпункты. <h1> — главный тайтл игры, дальше — <h2>, <h3> и так далее до <h6>. 

<h1>Гайд по выживанию в The Binding of Isaac</h1>
<h2>1. Основные типы комнат</h2>
<h3>1.1. Обычные комнаты</h3>
РЕЗУЛЬТАТ:

Гайд по выживанию в The Binding of Isaac

1. Основные типы комнат

1.1. Обычные комнаты

Текст, выделение, переносы

Для обычного текста используется тег <p>. Например, можно написать описание билда в Baldur’s Gate 3 или пояснение к ферме в Stardew Valley. <strong> — подчеркнуть важность, <em> — акцент. 

<p>
  Это <strong>важный билд мага</strong> для Baldur's Gate 3,
  а это <em>рекомендация для новичков</em>.
</p>

<p>Перенос строки внутри текста<br>например, для списка бонусов.</p>
РЕЗУЛЬТАТ:

Это важный билд мага для Baldur's Gate 3, а это рекомендация для новичков.

Перенос строки внутри текста
например, для списка бонусов.

Для подстрочных и надстрочных индексов используются <sub> и <sup>. Можно, например, указать урон заклинания или уровень предмета. 

<p>Урон заклинания: 2d6 + Int<sup>мод</sup></p>
<p>Зелье лечения уровня: H<sub>2</sub>O + травы</p>
РЕЗУЛЬТАТ:

Урон заклинания: 2d6 + Intмод

Зелье лечения уровня: H2O + травы

Цитаты и выделения

<blockquote>
  «Не копай прямо под себя».
  <br><small>— каждый выживший игрок в Minecraft</small>
</blockquote>

<p>
  Тег <mark> подсвечивает подсказку:
  <mark>всегда берите поливочную лейку в Stardew Valley</mark>.
</p>
«Не копай прямо под себя». — каждый выживший игрок в Minecraft

Тег <mark> подсвечивает фразу, как важную подсказку в интерфейсе: всегда берите поливочную лейку в Stardew Valley.

раздел 3

Часто используемые HTML-теги

Представь, что каждый тег — это отдельный тип блока интерфейса: панель инвентаря, карточка квеста, окно настроек звука и т.п.

Теги структуры и оформления

<div> Блочный контейнер. Как отдельная панель инвентаря в игре. 
<div class="inventory">
  <h2>Инвентарь героя</h2>
  <p>Ключ от подземелья Деспота.</p>
</div>
<span> Строчный контейнер — подсветить часть текста (например, редкий предмет). 
<p>Выпал предмет:
  <span class="legendary">Меч Криперов</span>
</p>
<header> Шапка страницы или раздела — как верхняя панель меню игры. 
<header>
  <h1>Меню Baldur's Gate</h1>
</header>
<footer> Подвал — как панель с копирайтом и версиями патча. 
<footer>
  <p>v1.0. Патч: «Stardew Valley Guide».</p>
</footer>

Семантические теги

<nav> Навигация — как меню выбора: «Новая игра», «Продолжить», «Настройки». 
<nav>
  <a href="#isaac">Isaac</a>
  <a href="#minecraft">Minecraft</a>
  <a href="#stardew">Stardew Valley</a>
</nav>
<main> Основной контент страницы — сам «игровой экран». 
<main>
  <h2>Гайд по крафту</h2>
  <p>Как сделать алмазную броню в Minecraft.</p>
</main>
<section> Смысловой раздел — например, блок «Квесты». 
<section id="quests">
  <h2>Активные квесты</h2>
</section>
<article> Самостоятельный материал — отдельный гайд по игре. 
<article>
  <h2>Как пережить первую ночь в Minecraft</h2>
  <p>Пару советов...</p>
</article>

Теги для подсказок и сокращений

Тег <abbr> помогает показывать расшифровку сокращения при наведении, как всплывающий тултип. 

<p>
  <abbr title="HyperText Markup Language">HTML</abbr>
  — язык разметки.
</p>
РЕЗУЛЬТАТ:

HTML — язык разметки.

раздел 9

CSS-селекторы: как выбрать элемент

Селектор — это «адрес» элемента, к которому применяются стили.

По тегу (типу)

p {
  color: #111827;
}

По классу

.game-card {
  border: 2px solid #a855f7;
  border-radius: 12px;
  padding: 8px;
}
<div class="game-card">
  <h3>Stardew Valley</h3>
  <p>Спокойная ферма про жизнь в деревне.</p>
</div>
РЕЗУЛЬТАТ:

Stardew Valley

Спокойная ферма про жизнь в деревне.

По идентификатору (id)

#minecraft-guide {
  background-color: #ecfccb;
}
<article id="minecraft-guide">
  <h2>Гайд по выживанию в Minecraft</h2>
</article>

Группировка селекторов

h1, h2, h3 {
  font-family: "Segoe UI", sans-serif;
}
раздел 10

CSS-комбинаторы: связи между элементами

Комбинаторы помогают уточнить, где именно находится элемент — как если бы ты искал нужную кнопку только в меню паузы, а не по всей игре.

1. Потомок (пробел)

main p — все абзацы внутри <main>. 

main p {
  line-height: 1.6;
}
<main>
  <p>Этот абзац внутри main, стиль применится.</p>
</main>

<p>Этот абзац вне main, стиль не применится.</p>
Если написать .class1.class2 без пробела — это не комбинатор, а один элемент с двумя классами: class="class1 class2".

2. Прямой ребёнок >

ul > li — только элементы <li>, которые являются ПРЯМЫМИ детьми <ul>. 

ul > li {
  list-style-type: square;
}
<ul>
  <li>Этот пункт — прямой ребёнок ul, стиль применится</li>
  <li>
    Вложенный пункт:
    <ul>
      <li>А здесь уже другой уровень</li>
    </ul>
  </li>
</ul>

3. Соседний элемент +

h2 + p — первый абзац <p> сразу после <h2>. 

h2 + p {
  font-weight: bold;
}
<h2>Раздел</h2>
<p>Этот абзац сразу после h2, будет жирным.</p>
<p>А этот уже нет.</p>

4. Все следующие соседи ~

h2 ~ p — все абзацы <p>, которые идут после <h2> на том же уровне вложенности. 

h2 ~ p {
  color: #6f42c1;
}
<h2>Тема</h2>
<p>Абзац 1 после h2 — стиль есть.</p>
<p>Абзац 2 после h2 — стиль тоже есть.</p>
